## Documentation Standards

### Critical Rules
- **CommonMark Strict Compliance** — ALL documentation MUST follow CommonMark specification exactly.
- **NO TIME ESTIMATES** — NEVER document time estimates, durations, or completion times unless explicitly asked.

### CommonMark Essentials
- **Headers:** ATX-style ONLY (`#`), single space after `#`, hierarchical order (don't skip levels)
- **Code Blocks:** Fenced blocks with language identifier
- **Lists:** Consistent markers, proper indentation for nesting, blank line before/after
- **Links:** Inline `[text](url)` or reference style. No bare URLs without `<>` brackets
- **Line Breaks:** Two spaces + newline, or blank line between paragraphs

### Style Guide Principles
- **Task-Oriented:** Write for user GOALS, not feature lists
- **Clarity:** Active voice, present tense, direct language, second person
- **Structure:** One idea per sentence, one topic per paragraph
- **Accessibility:** Descriptive link text, semantic heading hierarchy

### Mermaid Diagrams
- Always specify diagram type first line
- Use valid Mermaid v10+ syntax
- Keep focused: 5-10 nodes ideal, max 15
- Types: flowchart, sequenceDiagram, classDiagram, erDiagram, stateDiagram-v2, gitGraph
