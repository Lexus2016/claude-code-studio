**Step 2: Add _buildReplyKeyboard method to TelegramBot class**

```javascript
_buildReplyKeyboard(ctx) {
  const row1 = [];

  // Write button always first — includes chat name when session active
  if (ctx.sessionId) {
    let chatName;
    try {
      const sess = this.db.prepare('SELECT title FROM sessions WHERE id=?').get(ctx.sessionId);
      chatName = (sess?.title || this._t('chat_untitled')).substring(0, 18);
    } catch {
      chatName = this._t('chat_untitled');
    }
    row1.push({ text: `${this._t('kb_write')} · ${chatName}` });
  } else {
    row1.push({ text: this._t('kb_write') });
  }
  row1.push({ text: this._t('kb_menu') });

  const rows = [row1];

  // Second row: project context + status
  if (ctx.projectWorkdir) {
    const pName = ctx.projectWorkdir.split('/').filter(Boolean).pop() || '...';
    rows.push([
      { text: `${this._t('kb_project_prefix')} ${pName}`.substring(0, 28) },
      { text: this._t('kb_status') },
    ]);
  } else {
    rows.push([{ text: this._t('kb_status') }]);
  }

  return {
    keyboard: rows,
    resize_keyboard: true,
    is_persistent: true,
  };
}
```

**Step 4: Update _showScreen to accept optional reply keyboard**

Add an optional parameter to _showScreen for when we want to also set the reply keyboard:

```javascript
async _showScreen(chatId, userId, text, keyboard, { replyKeyboard } = {}) {
  const params = {};
  if (keyboard) {
    params.reply_markup = JSON.stringify({ inline_keyboard: keyboard });
  } else if (replyKeyboard) {
    params.reply_markup = JSON.stringify(replyKeyboard);
  }
  const sent = await this._sendMessage(chatId, text, params);
  return sent;
}
```

Note: Only one reply_markup type per message. When inline keyboard is present, the reply keyboard cannot be updated on that same message. The `replyKeyboard` param is used only when there's NO inline keyboard.
